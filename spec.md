# 殴性格診断 仕様書

## 概要
友達グループ内で「あなたは○○タイプ!」と診断される、身内ノリ全開の性格診断Webアプリ。
タイトルは「殴性格診断」。
8問の質問に答えると、5人のメンバーの中から最も近いタイプが診断される。

## 診断対象メンバー & 特徴タグ

| メンバー | 特徴タグ |
|---|---|
| 島だけさん | マイペース / のばまん好き / 話たがり / 実は豆腐メンタル / 一人プレイが苦手で聞き役が欲しい |
| ミナノさん | 冷笑 / 斜に構えてる / 殴るの楽しい / 意味が分からない |
| ピリンチさん | ミステリアス / 静か / かわいい(男) / ハマったゲームにのめりこむ |
| ぐらにうさん | 善良なキチガイ(予測不能・悪気のないマイペースな暴走) |
| 面真さん | 悪質なキチガイ(計算高く、狙って人をいじる) |

## 質問リスト(全8問・4択)

### Q1. ゲームで負けた時、どうする?
- 表面は笑ってるけど内心かなり凹む → 島だけさん
- 相手を煽ってさらに攻撃する → ミナノさん
- 黙って無言でリベンジに燃える → ピリンチさん
- ルールを都合よく捻じ曲げて言い訳する → 面真さん

### Q2. 誰かが変なことを言い出した時、どうする?
- 便乗してさらにおかしなことを言い出す → 面真さん
- 鼻で笑って一言で切り捨てる → ミナノさん
- 無言で真顔のまま聞いている → ピリンチさん
- 予想の斜め上のリアクションを返す → ぐらにうさん

### Q3. Minecraftでの遊び方は?
- 建築に没頭して喋りながら長時間作業 → 島だけさん
- 他人が作ったものを面白半分で壊す → ミナノさん
- 黙々と一人で採掘や探索に没頭する → ピリンチさん
- ルールも常識も無視した建築をする → 面真さん

### Q4. ゲームは一人でやる派?みんなでやる派?
- 誰かに実況しながらじゃないと物足りない → 島だけさん
- 一人で黙々とやる方が好き → ピリンチさん
- 誰かを巻き込んで振り回すのが好き → ミナノさん
- そもそも「一人」という概念が謎 → ぐらにうさん

### Q5. Discordのvc、気づいたら何時間経ってる?
- ずっと喋り続けて気づいたら深夜になってる → 島だけさん
- ほとんど喋らないけど誰よりも長くいる → ピリンチさん
- 気分次第でふらっと来てふらっと落ちる → ぐらにうさん
- 変なタイミングで乱入してきて空気を壊す → 面真さん

### Q6. 殴ることについてどう思う?
- 楽しくてついやってしまう、加減はあまり考えない → ミナノさん
- 悪気なく笑いながら地雷を踏み抜く、結構ガチで痛い → ぐらにうさん
- ちゃんと一番痛いところを計算して狙ってくる → 面真さん
- 表面上は笑ってるけど、実はそこそこ本気で痛がってる → 島だけさん

### Q7. 遊戯王で一番好きなデッキタイプは?
- 罠をじっくり仕込んで、気づいたら詰んでるのが気持ちいい → ミナノさん(ラビュリンス)
- 長期戦でじっくり展開しながら、喋りつつ圧倒するのが好き → 島だけさん(軍貫)
- とにかく火力全開、真っ向勝負でぶっ飛ばすのが好き → ぐらにうさん(サイバードラゴン)
- ふわふわマイペースに動かしてたら、いつの間にか勝ってた → ピリンチさん(ふわんだりぃず)

### Q8. 興味ない話・困った話をどう流す?
- 一応相槌打ちながら聞くけど、実はほぼ聞いてない → ピリンチさん
- 露骨に「あーはいはい」で流して次の話題に変える → ミナノさん
- 話を無理やり自分の得意分野に持っていく → 島だけさん
- そもそも話が流れてることに気づかず突っ走る → ぐらにうさん

## スコアリングロジック
- 各選択肢はメンバー1人に1ポイントを付与
- 8問終了後、最も合計ポイントが高いメンバーを診断結果とする
- 同点の場合はランダムで1人を選出(またはどちらも表示、実装時に判断)

## 結果画面の仕様
- 一致度(%)は表示しない。結果のみをシンプルに表示
- 「あなたは〇〇さんタイプ!」という結果 + そのメンバーの特徴を一言コメントとして表示
- 2位・3位などの表示は不要(今回はスコープ外)

## 共有機能(Discord向け)
結果画面に以下のボタンを設置する。

### ① テキストコピー
- 「結果をコピー」ボタンを押すと、診断結果のテキストがクリップボードにコピーされる
- コピーされる文面の例:
  ```
  【殴性格診断】あなたは「〇〇さん」タイプでした!
  ```
- `navigator.clipboard.writeText()` を使用

### ② 画像として生成・コピー
- 「画像をコピー」ボタンを押すと、結果画面(タイトル・診断結果・キャッチコピーなどをまとめたカード状のデザイン)を画像化し、クリップボードにコピーする
- Discordのチャンネルにそのまま `Ctrl+V` / `Cmd+V` で貼り付けられることが目的
- 実装方針:
  - 結果表示用のカードDOM要素を用意し、`html2canvas`(CDN経由で読み込み)などでcanvas化 → `canvas.toBlob()` → `navigator.clipboard.write([new ClipboardItem({ 'image/png': blob })])` でクリップボードに画像として書き込む
  - `navigator.clipboard.write` の画像コピーに対応していないブラウザ(一部のSafari/モバイル等)向けに、フォールバックとして「画像をダウンロード」ボタン(`canvas.toBlob` → `<a download>`)も用意する
  - カードのデザインはガチャ・カオス感のあるテイストに合わせ、背景色・診断結果・簡単なひとことコメントを含める(正方形または横長の、Discordに貼っても見やすい比率で作成)
- コピー/保存成功時は「コピーしました!」等のフィードバックをUI上に一瞬表示する

## 今回のスコープ外(将来的な拡張候補)
- 名前入力によるイースターエッグ(特定の名前を入れると固定セリフが出る仕組み)は今回は実装しない。後日追加できるよう、名前入力欄自体は用意してもよいが、分岐ロジックは不要

## デザインテイスト
- 「少しガチャ・カオス感のあるデザイン」
- ポップさとカオスさを両立させるイメージ(例: 派手な配色、ランダムに動く/揺れる要素、大げさな演出、ガチャガチャ感のあるフォントやエフェクト)
- 結果発表時にちょっとした演出(アニメーションやガチャっぽい効果音・画面揺れなど)があると盛り上がる

## 技術構成
- HTML / CSS / JavaScript の単一ファイル構成(ビルド不要、依存関係なし)
- 質問・選択肢・スコアリングロジックはJavaScript内にデータとして定義し、後から質問や選択肢を追加・編集しやすい構造にする
- スマホでの閲覧も想定し、レスポンシブ対応必須(LINE等で友達に送って開かれる想定のため)

## 公開方法
- GitHub Pages または Vercel でのホスティングを想定
- 静的ファイルのみで完結するため、上記いずれかにデプロイしてURLを発行し、友達にURLを共有する運用
